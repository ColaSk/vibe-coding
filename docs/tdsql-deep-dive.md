## 1. 概述与背景

TDSQL（Tencent Distributed SQL）是腾讯云自研的企业级分布式数据库产品体系，定位为**金融级高可用、强一致、可水平扩展**的关系型数据库。其发展历程超过十年，从腾讯内部海量业务（微信支付、QQ 等）的实践中锤炼而来，2014 年被微众银行选中作为核心交易数据库，2015 年在腾讯云正式商用。

截至 2026 年，TDSQL 累计服务超 **10 万家企业**，落地超 **100 家金融机构核心系统**，覆盖四大国有银行主力数据库，中国十大银行中七家选择 TDSQL。在 IDC 发布的《中国金融行业分布式事务型数据库市场份额，2024》中，TDSQL 以 **21.32%** 的市场份额蝉联金融行业第一，银行子市场以 **22.48%** 位居榜首。

TDSQL 已形成覆盖多种场景的完整产品矩阵：

| 产品 | 定位 | 架构 | 兼容性 | 核心场景 |
|------|------|------|--------|----------|

## 3. 核心架构与原理

### TDSQL MySQL 版（经典分布式架构）

基于存算分离的 Share-Nothing 架构，核心三层：

```
客户端
  ↓
计算层（SQL Engine / TXSQL 内核）
  ├── SQL 解析 / 优化器 / 执行器
  ├── 分布式事务管理（2PC）
  └── 连接池 / 路由管理
  ↓
存储层（Data Node / Set）
  ├── InnoDB-like 存储引擎
  ├── Raft/半同步复制（一主多备）
  └── 数据分片（Hash Sharding）
  ↓
管控层（ZooKeeper + Scheduler + Agent）
  ├── 元数据管理
  ├── 故障检测与自动切换
  └── 数据调度与负载均衡
```

- **TXSQL 内核**：腾讯自研的 MySQL 分支（约 300 万行代码增量），完全兼容 MySQL 语法和 API
- **数据分片**：通过 Proxy（TGW/SQLEngine）自动路由，业务无感知
- **强同步复制**：基于 Raft 或增强半同步，确保 RPO=0
- **分布式事务**：两阶段提交（2PC），全局一致性快照读

### TDSQL Boundless（新一代架构）

采用自研 **TDStore** 存储引擎，三大核心组件：

| 组件 | 角色 | 关键特性 |
|------|------|----------|
| **SQLEngine** | 计算层 | 基于 MySQL 8.0，多主无状态，每节点均可读写 |
| **TDStore** | 存储层 | LSM-Tree + Multi-Raft KV 引擎，数据按 Key 范围分布在 Region 上 |
| **TDMC** | 元数据 | Raft 一主两备集群，分配全局事务 ID，协调数据调度 |

**对等架构（HyperNode）**：计算 + 存储 + 日志合并在单个节点内，本地访问优先，远端 RPC 补充，兼顾性能与弹性。

### TDSQL-C（云原生架构）

存算分离，计算节点只写 Redo 日志到分布式存储层：

- IO 减少 60%+，写入性能提升 90%+
- 主从基于 Redo 同步，延迟 ms 级
- 计算节点无状态，秒级故障切换、秒级升配
- 最高 128TB 存储，单点百万 QPS

## 4. 关键特性详解

### 4.1 金融级高可用

| 特性 | 说明 |
|------|------|
| 强同步复制 | Raft / 增强半同步，RPO=0 |
| 自动故障切换 | 秒级主备切换，RTO < 30s |
| 多中心容灾 | 支持两地三中心、两地四中心部署 |
| 数据一致性 | 99.9999999%（9 个 9）数据可靠性 |
| 服务可用性 | 99.99%+ |

### 4.2 水平扩展

- 在线弹性扩缩容，无需停机
- 线性扩展比率 84%（TPC-C 测试验证）
- TDSQL Boundless 支持数据自动分裂、合并、迁移

### 4.3 MySQL/PG/Oracle 兼容

- MySQL：100% 协议兼容，基于 TXSQL 内核
- PostgreSQL：PG 模式完全兼容，Oracle 模式语法兼容 98%+
- 2025 年一体化版本实现一次部署多协议切换

### 4.4 智能运维（扁鹊平台）

- 200+ 性能指标毫秒级监控
- 故障定位效率提升 80%
- AI 驱动的索引优化，查询性能提升 40%
- 自动化备份、巡检、容量规划

### 4.5 迁移工具链（DBbridge）

- 支持 Oracle → TDSQL、MySQL → TDSQL 等异构迁移
- 自动 DDL/DML 转换
- 代码修改量压缩至 5% 以内
- 支持在线不停服迁移

### 4.6 多模态能力（TDSQL Boundless，2026 新增）

- 分布式存储层同时支持：结构化数据、JSON 文档、全文检索、向量检索
- 单一引擎覆盖多种数据模型需求

## 10. 适用场景与建议

### 推荐使用的场景

- **金融核心交易系统**：银行核心、证券交易、保险理赔等对强一致、高可用要求极高的场景
- **Oracle/MySQL 迁移**：借助 DBbridge 和高兼容性实现低成本迁移
- **腾讯云用户**：与腾讯云生态深度集成，开箱即用
- **高弹性互联网业务**：TDSQL-C Serverless 按需扩缩，适合流量波动大的场景
- **PB 级大规模数据**：TDSQL Boundless 的 LSM-Tree 压缩和多模态能力

### 不推荐的场景

- **追求开源和社区生态**：TiDB 在开源社区活跃度和全球影响力上更优
- **多云/混合云灵活部署**：OceanBase（OB Cloud）或 TiDB 的跨云能力更强
- **轻量级单机场景**：普通 MySQL 即可满足，TDSQL 的分布式架构在小规模下成本偏高
- **非关系型纯文档/图数据库需求**：虽有 Boundless 的多模态，但成熟度不及专用 NoSQL

### 注意事项

- 产品线较多（TDSQL MySQL 版 / TDSQL-C / TDSQL Boundless / TDSQL PG 版），选型前需明确业务场景匹配
- 私有化部署建议提前联系腾讯获取 License 和软件包
- 从 Oracle 迁移时，PG 版的 Oracle 兼容模式优于 MySQL 版
- 建议通过 POC 验证实际业务负载下的性能表现
